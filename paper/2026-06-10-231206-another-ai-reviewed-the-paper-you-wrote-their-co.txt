      466 -\subsubsection{Physics-aware data augmentation for the CNN}   
      467 -The cubic simulation grid has the full $O_h$ octahedral symmet
          -ry group                                                      
      468 -of 48 operations. However, the UV field illuminates the cube a
          -long the                                                      
      469 -$+z$ axis, breaking any operation that interchanges $z$ with a
          - transverse                                                   
      470 -axis. The physically valid augmentations are the eight $z$-pre
          -serving                                                       
      471 -operations (four rotations about $z$ and four reflections thro
          -ugh planes                                                    
      472 -containing $z$). Scalar fields (density, temperature, etc.) tr
          -ansform                                                       
      473 -by index permutation. Polar vector fields (velocity) transform
          - as                                                           
      474 -$\bm{v}' = \bm{R}\bm{v}$. Axial vector fields (magnetic field)
      475 -transform as $\bm{B}' = \det(\bm{R})\,\bm{R}\bm{B}$, with addi
          -tional                                                        
      476 -left--right face index swaps on axes whose sign is negated by 
          -$\bm{R}$.                                                     
      477 -Using the full 48-element group injects physically invalid tra
          -ining                                                         
      478 -samples and was verified to degrade results.                  
      456 +Data augmentation respects the broken symmetry of the illumina
          +ted cube:                                                     
      457 +of the 48 octahedral operations, only the eight $z$-preserving
          + ones                                                         
      458 +(rotations about $z$ and reflections through planes containing
          + $z$) are                                                     
      459 +physically valid. Scalars transform by index permutation, velo
          +cities as                                                     
      460 +polar vectors, and the face-centred magnetic field components 
          +as axial                                                      
      461 +vectors with the appropriate face swaps. Using all 48 operatio
          +ns                                                            
      462 +injects unphysical samples and degraded validation accuracy in
      463 +development runs. Each fold therefore trains on               
      464 +$6 \times 8 = 48$ augmented volumes.

      179 +                         chunk: int = 2_000_000) -> tuple[np.n
          +darray, np.ndarray]:                                          
      180 +    """Memory-efficient per-column standardization (float32, i
          +n place).                                                     
      181 +                                                              
      182 +    Equivalent to StandardScaler fit on X_tr / transform on bo
          +th, but avoids                                                
      183 +    sklearn's full-size float64 temporaries (which need >5 GB 
          +for the                                                       
      184 +    14.7M x 56 spatial matrix).  Means/variances are accumulat
          +ed in float64                                                 
      185 +    over row chunks; the arrays are modified in place.        
      186 +    """                                                       
      187 +    n, d = X_tr.shape                                         
      188 +    s  = np.zeros(d, dtype=np.float64)                        
      189 +    s2 = np.zeros(d, dtype=np.float64)                        
      190 +    for i in range(0, n, chunk):                              
      191 +        c = X_tr[i:i + chunk].astype(np.float64)              
      192 +        s  += c.sum(axis=0)                                   
      193 +        s2 += (c * c).sum(axis=0)                             
      194 +    mu = (s / n)                                              
      195 +    sd = np.sqrt(np.maximum(s2 / n - mu * mu, 0.0))           
      196 +    sd[sd < 1e-8] = 1.0                                       
      197 +    mu32 = mu.astype(np.float32)                              
      198 +    sd32 = sd.astype(np.float32)

      201 +        for i in range(0, m, chunk):

      203 +            arr[i:i + chunk] /= sd32                          
      204 +    return X_tr, X_va
